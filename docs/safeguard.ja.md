# コマンドセーフガード

FloatingMacro はターミナルへのテキスト挿入やコマンド実行をボタン一発で行えます。
これは AI エージェントが利用する場合、意図せず危険なコマンド（例: `rm -rf`）が実行されるリスクを生みます。
**コマンドセーフガード**はそのリスクを軽減するための機能です。

---

## 仕組み

コマンドやテキストがターミナルに送られる直前に、登録した **禁止パターン** との照合が行われます。

| 状況 | 動作 |
|---|---|
| パターンに一致しない | そのまま実行 |
| パターンに一致した | **確認ダイアログ**を表示して、ユーザーが判断 |
| オートパイロットモードが有効 | パターンに一致しても確認なしで実行 |

チェック対象はすべての実行経路です。

- ボタンをクリックして実行する **terminal アクション**（コマンド文字列）
- ボタンをクリックして実行する **text アクション**（ターミナルに貼り付けるテキスト）
- マクロ（複数ステップ）内のサブアクション
- Control API（`POST /action`）経由でAIが送ってくるコマンド

いずれの経路を通っても、同じルールが適用されます。

---

## 設定を開く

メニューバーのアイコン → **「ボタン編集…」** → ウィンドウ上部の **「セキュリティ」タブ**

---

## 確認ダイアログ

パターンに一致するコマンドが実行されようとすると、次のようなダイアログが表示されます。

```
危険なコマンドが検出されました

禁止パターン「rm -rf」が含まれています。

rm -rf /some/path

本当に実行しますか？

[ キャンセル ]  [ 実行する ]
```

- **「キャンセル」**（デフォルト、Return キーで選択） → コマンドを中止します
- **「実行する」** → そのまま実行します

> **注意**: 「キャンセル」がデフォルトボタンです。Enter を押し間違えても止まります。

---

## 禁止パターン一覧

パターンは**大文字・小文字を区別しない部分一致**で判定します。
`rm -rf` を登録すると `RM -RF /home` も引っかかります。

### デフォルトで登録されているパターン

| パターン | 想定する危険操作 |
|---|---|
| `rm -rf` / `rm -fr` | ディレクトリの再帰削除 |
| `sudo rm` | root 権限での削除 |
| `> /dev/` | デバイスへの直接書き込み |
| `dd if=/dev/` | デバイスからのディスクコピー・破壊 |
| `mkfs` | ファイルシステムの再フォーマット |
| `:(){ :|:& };:` | Fork Bomb |
| `chmod -R 777` / `chmod 777 /` | 権限の一括開放 |
| `sudo chmod` | root 権限での権限変更 |
| `shred ` | ファイルの安全消去 |
| `wipefs` | パーティションシグネチャの消去 |
| `diskutil eraseDisk` / `diskutil zeroDisk` | macOS ディスク初期化 |
| `format c:` | Windows ドライブフォーマット |

### パターンを追加・編集・削除する

「セキュリティ」タブの **「確認対象パターン一覧」** で管理できます。

**追加**
: テキストフィールドにパターンを入力して「追加」または Enter。

**編集**
: 行の「編集」ボタン → 内容を書き換えて「確定」。

**削除**
: 行のゴミ箱ボタン。

**デフォルトに戻す**
: 「デフォルトパターンに戻す」ボタンで上の15件に一括復元。

変更は即座に保存されます（アプリ再起動後も維持）。

---

## セーフガードを無効にする

「確認ダイアログを有効にする」トグルを OFF にすると、パターン照合自体が行われなくなります。
**AI に完全に操作を委ねたい場合でも、こちらではなくオートパイロットモードの使用を推奨します。**
オートパイロットはパスワードによる保護があるため、誤操作での無効化を防げます。

---

## オートパイロットモード

AI エージェントに完全な自律実行を許可したいときのモードです。
このモードが有効な間は、禁止パターンに一致するコマンドでも確認ダイアログなしで実行されます。

**有効化にはパスワードが必要**です。これにより、AI がプログラムから勝手にモードを切り替えることを防ぎます。

### パスワードを設定する（初回）

1. 「セキュリティ」タブ → オートパイロットセクションの **「パスワードを設定する…」**
2. パスワードを2回入力して **「設定する」**
   - 4文字以上必須
   - パスワード本体は保存されません。SHA-256 ハッシュだけが `config.json` に記録されます。

### オートパイロットを有効にする

1. **「オートパイロットを有効にする…」** をクリック
2. パスワードを入力して **「確認」**
   - 正しければ：セクション枠がオレンジ色になり「有効」バッジが表示されます
   - 誤りの場合：「パスワードが違います」と表示され、モードは変化しません

> **有効中はセクション全体がオレンジ色**になるため、意図せず有効になっていることに気づきやすくなっています。

### オートパイロットを無効にする

**「オートパイロットを無効にする」** をクリックするだけです。パスワードは不要です。

> 作業が終わったら**必ず無効に戻してください**。

### パスワードを変更する

1. **「パスワードを変更する…」** をクリック
2. **現在のパスワード** と **新しいパスワード** を入力して **「変更する」**
   - 現在のパスワードが違う場合はエラーになります
   - 変更が完了するとオートパイロットは **自動的に無効化** されます（再有効化には新しいパスワードが必要）

---

## セーフガードの優先順位

複数の設定が重なった場合の動作です。

```
autopilotEnabled = true   →  何もチェックしない（全コマンド実行）
        ↓
enabled = false           →  何もチェックしない（全コマンド実行）
        ↓
パターン不一致            →  そのまま実行
        ↓
パターン一致              →  確認ダイアログ
        ↓
ユーザーが「キャンセル」  →  実行中止（エラーバナーが表示される）
```

## config.json の対応フィールド

設定は `~/Library/Application Support/FloatingMacro/config.json` に保存されます。

```json
{
  "commandBlacklist": {
    "enabled": true,
    "patterns": [
      "rm -rf",
      "sudo rm",
      "..."
    ],
    "autopilotEnabled": false,
    "autopilotPasswordHash": "sha256ハッシュ（64桁の16進数）"
  }
}
```

| フィールド | 説明 |
|---|---|
| `enabled` | `false` にするとパターン照合をスキップ |
| `patterns` | 照合する文字列のリスト |
| `autopilotEnabled` | `true` のとき確認ダイアログをスキップ |
| `autopilotPasswordHash` | パスワードの SHA-256 ハッシュ。`null` のときオートパイロットは有効化不可 |

> **注意**: `autopilotEnabled` を `config.json` に直接 `true` と書いても有効になります。ただしパスワードハッシュが `null` のままだとパスワード保護がない状態になるため、UI からの設定を推奨します。

## よくある質問

**Q. AI が勝手にオートパイロットを有効にできますか？**

Control API（`POST /settings/...`）は設定変更に対応していますが、`autopilotEnabled` をプログラムから直接 `true` にするエンドポイントは提供していません。UI 経由のパスワード確認が唯一の有効化手段です。

**Q. パスワードを忘れました。**

パスワード本体は保存されていないので復元できません。`config.json` を直接編集して `"autopilotPasswordHash": null` にリセットすれば、初回設定から再開できます。その際、`"autopilotEnabled": false` も確認してください。

**Q. マクロの途中でダイアログが出るとその後のステップはどうなりますか？**

「キャンセル」を選ぶと `stopOnError: true`（デフォルト）の場合はマクロ全体が中断されます。`stopOnError: false` の場合は問題のステップだけスキップして続行します。

**Q. text アクションなのにダイアログが出ました。**

text アクションのテキストにも同じパターン照合が行われます。クリップボード経由でテキストを挿入する際、フォーカス中のウィンドウがターミナルであれば実行される可能性があるためです。
